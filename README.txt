ゲボファミ

ゲボファミはファミコン内蔵音源や拡張音源機能を持つファミコンカセットをUSB接続の音源ユニットにする為の物体です。
Windows PCからゲボファミを制御して各音源に対応したVGM/VGZファイルを演奏することが出来ます。
現時点の対応音源は内蔵APU, FDS、MMC5, VRC6, VRC7, N163です。
MMC5, VRC6, N163対応のVGM/VGZファイルは存在していない筈なのでここにあるnezplug(NEZplug++ VGM mod改造版)
を使用してNSFファイルから変換する必要があります。
NSFファイルの楽曲はほとんどが無限ループ楽曲なので変換すると尻切れトンボの曲データとなります。
VGMRipsにあるようなループ再生するようなVGMデータは作成できません。これで我慢して下さい。
初期バージョンの赤白ファミコンのみで動作確認していますそれ以外は知りません。

* 作り方

部品リスト
C1 0.47u～10u 電解
C2 0.47u～10u 電解
C3 0.1u セラミック
J1 エッジコネクタ60P
J2 3.5mmステレオミニジャックPJ-307 必要に応じて加工(mod1.jpg mod2.jpgを参考にして下さい)
JP1 部品はありません後述 必ずジャンパして下さい
R1 1k
R2 1k
U1 RaspberryPiPico
U2 74xx245又は74xx541 (xxはLS/HC/HCTの何れか)

見ての通りの簡単なものですがピコボード上のLEDとダイオードを外す必要があります。
-> 取り外し済の状態でお送りしますので取り外し作業は不要です。
ファームウェア書込み済の状態でお送りしますのでピコボードに書込みを行なわないで下さい。

JP1はU2が74xx245の場合は中央と245側のランドを半田でジャンパ(ショート)して下さい。
74xx541の場合は中央と541側のランドを半田でジャンパして下さい。

御注意
回路を簡素にする為に信号電圧レベル変換回路を省略しています
RP2040のIOピン(プラス側にダイオードが入っていないピンのみ)をファミコンのスロット端子に直結しています。
実質的に問題ないであろう事を確認していますが定格外の使い方をしているのでその辺自己責任でお願いします。

* セッティング

ファミコンにゲボファミを装着し、必要に応じで拡張音源付きのカセットをゲボファミに挿してください。
くれぐれも差し込む向きを間違えないように注意して下さい。ゲボファミとWindwos PCをUSBケーブルで接続、
ファミコンにACアダプターを接続し通電するとWindows PCがゲボファミをシリアルポートとして認識します。
ここ重要 >>>> 通電したら必ず一度ファミコンのリセットボタンを押してください <<<< これで準備完了。
Windowsコマンドプロンプトで演奏プログラム(nesplay)を実行するとゲボファミのステレオジャックから音が出るんだという。
勿論、ステレオジャックそのものからは音は出ません。ライン出力なのでアンプを接続して聞いて下さい。
左側からファミコン内蔵音源の音、右側からは拡張音源の音が出ます。

* 演奏プログラム(nesplay)の使い方

Windowsコマンドプロンプトから以下のように実行して下さい。

nesplay [-s] [-l count] comport file

例: nesplay -l 3 com1 song.vgz

comportはゲボファミのCOMポート、fileにはVGM又はVGZファイルを指定します。
ファミコン向けのファイルに限らずYM2413向けのファイルをVRC7で(不完全に)再生することも出来ます。
COMポート名はデバイスマネージャーで確認して下さい。

-sオプションは悪魔城伝説(VRC6)向けに作成されたファイルを魍魎戦記MADARAで再生(あるいはその逆)する場合に指定します。
-lオプションはループ再生パートの再生回数です。0以上の整数を指定して下さい。
-lオプション未指定の場合は1回の再生です。値に0を指定した場合は無限ループ再生となります
ループ再生パートの無いファイル(nezplugで変換したファイルもこれに含まれます)の場合はこのオプションは無視されます。

* ゲボファミの通信プロトコル

COMポートに以下のVGMコマンドを書き込むとゲボファミはコマンドに従ってファミコンシステムを制御します。

0xb4, address, data // 本体内蔵APU/FDS書込みコマンド
0x51, address, data // YM2413(VRC7)書込みコマンド
0x67, 0x66, 0xc2, size, size, size, size, address, address, (data) // 本体RAM書込みコマンド

上記3コマンドのみの対応です。それ以外のコマンドは書き込まないで下さい。
ゲボファミはコマンド受信中に100ミリ秒以上データが途絶えると受信中のコマンドを破棄して次のコマンドを待ちます。

0x67コマンドは本体RAMにデータを書き込む為のコマンドですがファミコン音源はアドレス空間にマップされているので
音源制御コマンドとしても使用出来ます。MMC5, VRC6, N163はこのコマンドで制御しています。

各コマンドの詳細は以下のページを参考にして下さい。
https://vgmrips.net/wiki/VGM_Specification

* COMポートプログラミングのヒント

通信はUSBで完結しているのでCOMポートの通信速度等の設定は意味を持ちません。デフォルト設定のままで大丈夫です。
